================================================================================
                    LUME - PROJECT SUMMARY & FEATURES LIST
================================================================================

PROJECT OVERVIEW:
-----------------
Lume is a global payroll platform built on the Stellar blockchain network that 
enables businesses to pay international teams with 90% lower fees compared to 
traditional methods like SWIFT and wire transfers. The platform provides instant 
cross-border payments with 5-second transaction finality.

TECH STACK:
-----------
- Frontend: Next.js 16.1.4 (React 19.2.3), TypeScript, Tailwind CSS 4
- Blockchain: Stellar SDK v13.3.0, Stellar Wallets Kit v1.9.5
- Database: Supabase (PostgreSQL with SSR)
- State Management: Zustand v5.0.10
- UI/Animation: Framer Motion v12.28.1, Lucide React v0.562.0
- Notifications: Sonner v2.0.7
- Charts: Recharts v3.7.0
- PDF Generation: jsPDF v4.0.0
- QR Code: qrcode v1.5.4
- External APIs: CoinGecko (XLM price data)

5. ESCROW & SECURE TRADE
------------------------
✅ 2-of-3 multisig escrow system (Buyer + Seller + Admin)
✅ Stellar-based secure vault accounts
✅ MY SALES TAB (For Sellers):
   - Create escrow order (Product ID + Amount in USDC)
   - Auto-generated Stellar keypair for each order
   - Order list with status badges (Created/Funded/Locked/Released/Refunded)
   - Copy escrow address functionality
   - Stellar Expert transaction links
   - Order status color coding with icons
✅ MARKETPLACE TAB (For Buyers):
   - Search orders by Product ID
   - Order detail view with amount and status
   - "Get Payment Address" - copy escrow vault address
   - "Lock Escrow" - locks funds with 2-of-3 multisig setup
   - "Confirm Receipt & Release Funds" - triggers release to seller
   - Freighter wallet signing integration
   - Automatic admin co-signing on release
✅ ESCROW FLOW:
   1. Seller creates order → Escrow vault generated
   2. Buyer searches order → Sends funds to vault
   3. Buyer locks escrow → Multisig setup applied
   4. Goods/services delivered → Buyer confirms receipt
   5. Buyer + Admin sign release → Funds sent to seller
✅ Security features:
   - Escrow secret keys stored server-side only
   - Public keys exposed, secrets never sent to client
   - 2-of-3 threshold prevents unilateral fund movement
   - Row-level security on Supabase database
